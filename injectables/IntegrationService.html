Description

This service provides methods to interact with payment integrations, including initializing payment intents for carts and orders using third-party payment providers like Stripe.

Usage

import { IntegrationService } from '@congarevenuecloud/ecommerce';

constructor(private integrationService: IntegrationService) {}

// or

export class MyService extends AObjectService {
private integrationService: IntegrationService = this.injector.get(IntegrationService);
}

calculateTax
calculateTax(businessObjectId: string, businessObjectType: string, shipToAddress: TaxAddress)

Calculates tax for a business object (Cart, Quote, or Order) using the integrated tax provider. This method sends a tax calculation request to the tax service integration and processes line items with tax amounts.

Example:

import { IntegrationService, TaxAddress } from '@congarevenuecloud/ecommerce';

export class MyComponent {
constructor(private integrationService: IntegrationService) {}

calculateTax(cartId: string) {
const address: TaxAddress = {
Line1: '123 Main St',
City: 'San Francisco',
Region: 'CA',
Country: 'US',
PostalCode: '94102'
};

this.integrationService.calculateTax(cartId, 'ProductConfiguration', address)
.subscribe((response) => {
});
}
}

See TaxAddress for address structure See TaxCalculationResponse for the structure of the response

Parameters :
Name Type Optional Description
businessObjectId string No

The unique identifier of the cart, quote, or order.
businessObjectType string No

The type of business object: 'ProductConfiguration' (Cart), 'Proposal' (Quote), or 'Order'.
shipToAddress TaxAddress No

The shipping address to use for tax calculation in TaxAddress format.
Returns : Observable<TaxCalculationResponse>

An Observable emitting a TaxCalculationResponse with a confirmation message.
getPaymentMetadata
getPaymentMetadata()

Retrieves payment metadata configuration including provider-specific settings like API keys. This method fetches configuration data needed to initialize payment providers on the client side.

Example:

import { IntegrationService, PaymentMetadata } from '@congarevenuecloud/ecommerce';
import { Observable } from 'rxjs/Observable';

export class MyComponent implements OnInit{
constructor(private integrationService: IntegrationService){}

getPaymentConfig(){
// First call makes API request, subsequent calls use cached data
this.integrationService.getPaymentMetadata().subscribe((metadata: PaymentMetadata) => {
// Use metadata to initialize payment UI
});
}
}
Returns : Observable<PaymentMetadata>

An Observable emitting PaymentMetadata configuration object containing provider settings.
getTaxMetadata
getTaxMetadata()

This method fetches configuration settings required for tax calculation integration, including the feature flag to determine if tax integration is enabled.

Example:

import { IntegrationService, TaxMetadata } from '@congarevenuecloud/ecommerce';
import { Observable } from 'rxjs';

export class MyComponent implements OnInit {
constructor(private integrationService: IntegrationService) {}

ngOnInit() {
// First call retrieves data from API, subsequent calls use cached response
this.integrationService.getTaxMetadata().subscribe((metadata: TaxMetadata) => {
if (metadata.EnableTaxIntegration) {
// Tax integration is enabled - initialize tax calculation features
} else {
// Tax integration is disabled
}
});
}
}

See TaxMetadata for the structure of the metadata response

Returns : Observable<TaxMetadata>

An Observable emitting a TaxMetadata configuration object containing provider-specific settings and feature flags.
initializePayment
initializePayment(businessObjectId: string, amount: number, currency: string, businessObjectType: string)

Creates a payment intent on the server for processing payments through integrated payment providers. This method initializes a payment session by communicating with the payment gateway API and returns the necessary credentials (client secret) to complete the payment on the client side.

Example:

import { IntegrationService } from '@congarevenuecloud/ecommerce';
import { Observable } from 'rxjs/Observable';

export class MyComponent implements OnInit{
constructor(private integrationService: IntegrationService){}

initializePayment(cartId: string, amount: number, currency: string){
this.integrationService.initializePayment(cartId, amount, currency).subscribe(response => {
// Use response.ClientSecret to initialize payment UI
});
}

// For Order payment
initializeOrderPayment(orderId: string, amount: number, currency: string){
this.integrationService.initializePayment(orderId, amount, currency, 'Order').subscribe(response => {
// Use response.ClientSecret to initialize payment UI
});
}
}

See PaymentIntentResponse for the structure of the response

Parameters :
Name Type Optional Description
businessObjectId string No

The unique identifier of the cart or order for which the payment intent is being created.
amount number No

The total payment amount in the base currency unit (e.g., dollars, euros). Will be converted to smallest currency unit by the payment provider.
currency string No

The three-letter ISO currency code (e.g., 'USD', 'EUR', 'GBP') representing the payment currency.
businessObjectType string No

The type of business object for the payment. Use 'ProductConfiguration' for cart-based payments or 'Order' for order-based payments.
Returns : Observable<PaymentIntentResponse>

An Observable emitting a PaymentIntentResponse containing the ClientSecret and paymentIntentId.
